# 文档同步

## 目的

确保项目文档（`docs/` 和 `AGENTS.md`）内部保持一致，交叉引用正确，描述准确。

## 使用时机

- 创建、移动或重命名文档文件后
- 重大架构变更可能导致现有文档失效后
- 声称文档工作已完成之前
- 被要求"审查文档"或"同步文档"时

## 流程

### 步骤 1：清单盘点

列出所有文档文件：

- `AGENTS.md`
- `docs/**/*.md`

### 步骤 2：链接审查

对每个 Markdown 文件：

1. 提取所有内部相对链接
2. 验证目标文件是否存在
3. 如链接失效，修复或删除该链接

### 步骤 3：内容一致性检查

对比各文档中的描述是否存在冲突：

- 模块名称和路径
- 版本号（JDK、Gradle 等）
- 示例中引用的文件路径
- 架构声明（如"平台无关"与实际依赖是否一致）

### 步骤 4：修复与验证

应用修复后，重新运行审查以确认问题已解决。

## 输出

报告检查了哪些内容，修复了哪些问题。
