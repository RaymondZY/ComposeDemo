# Use Cases — Story Input（故事输入框）

> 参考 UI：页面底部的"自由输入..."输入栏，含文字输入区、功能按钮区。

---

## 全局设计约束

- **焦点与光标**：输入框失去焦点时必须同步清除焦点（clear focus）并移除输入光标，确保收起后 UI 不再显示编辑态。
- **键盘联动控制**：禁用 Android 系统默认的 `adjustResize` / `adjustPan` 等自动调整页面行为。FeedScreen 内容的位移由应用层通过 IME Insets 精确控制，而非系统全局行为。
- **10dp 安全间距**：输入框底部与键盘顶部之间始终维持 ≥10dp 的间距。键盘升降过程中，FeedScreen 内所有内容作为一个整体联动平移。

---

## UC-01 默认收起键盘，点击展开

**参与者**：用户  
**前置条件**：输入框处于收起状态（键盘未弹出）  
**步骤**：

1. 用户点击输入框区域
2. 系统请求软键盘弹出
3. 输入框进入展开状态，用户可自由输入文字

**预期结果**：键盘弹出，输入框获得焦点，可接收文字输入

---

## UC-02 点击输入框外部收起键盘

**参与者**：用户  
**前置条件**：输入框处于展开状态（键盘已弹出）  
**步骤**：

1. 用户点击输入框以外的任意区域（StoryCard 背景等）
2. 系统收起软键盘
3. 输入框失去焦点，光标被移除（clear focus）
4. 输入框回到收起状态

**预期结果**：

- 键盘收起，输入框恢复收起状态
- 输入框内已有文字保留
- 输入框焦点被完全清除，不再显示编辑光标

## UC-03 输入框文字单行显示，超出省略

**步骤**：

1. 用户查看输入框区域

**预期结果**：输入框右侧显示两个按钮，从左到右依次为：

- **语音输入**按钮（麦克风图标）
- **加号面板**按钮（`+` 图标）

## UC-06 点击括号按钮在末尾插入括号并定位光标

**参与者**：用户  
**前置条件**：输入框处于展开状态，输入框内可有任意文字（含空）  
**步骤**：

1. 用户点击括号按钮
2. 系统在当前文字内容末尾追加一对全角括号 `（）`
3. 系统将光标移动到 `（` 与 `）` 之间

**预期结果**：文字末尾出现 `（）`，光标位于两个括号中间，用户可直接在括号内继续输入

## UC-09 有文字时加号按钮动态切换为发送按钮

**参与者**：用户  
**前置条件**：输入框处于任意状态  
**步骤**：

1. 用户在输入框中输入文字（内容非空）→ 加号按钮切换为**发送**按钮
2. 用户清空输入框内所有文字（内容为空）→ 发送按钮还原为**加号面板**按钮

**预期结果**：按钮图标随文字内容是否为空实时切换；发送按钮暂无实际功能（占位，后续迭代）

## UC-10 键盘弹出时 FeedScreen 内容与键盘保持 10dp 间距联动上移

**参与者**：系统  
**前置条件**：键盘从收起状态开始弹出；系统默认页面自动调整已禁用  
**步骤**：

1. 键盘开始从屏幕底部升起
2. 在键盘顶部与输入框底部之间的距离 **>** 10dp 时，FeedScreen 下所有内容保持不动
3. 当键盘继续上升，侵入输入框底部的 10dp 安全区时，键盘开始推动输入框及 FeedScreen 内所有上方 UI 元素整体上移
4. 上移位移量 = 键盘侵入 10dp 安全区的高度（即：键盘顶部到输入框底部的距离始终维持在 ≥10dp）

**预期结果**：

- 输入框底部与键盘顶部始终保持 ≥10dp 的间距
- FeedScreen 所有内容作为一个整体同步上移，无元素被键盘遮挡

---

## UC-11 键盘收起时 FeedScreen 内容联动还原

**参与者**：系统  
**前置条件**：键盘从展开状态开始收起；系统默认页面自动调整已禁用  
**步骤**：

1. 键盘开始从屏幕底部收起
2. FeedScreen 下所有内容随键盘下移，逐渐还原至原始既定位置
3. 当 UI 元素完全还原到既定位置后，停止移动
4. 键盘继续完全收起至屏幕外

**预期结果**：

- UI 元素先还原到位，键盘后完全收起
- 还原完成后，FeedScreen 所有内容位置与键盘弹出前完全一致
