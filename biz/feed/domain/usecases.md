# Feed 模块功能需求

## 概述

Feed 模块采用卡片滑动交互。本文件描述下拉刷新、加载更多及相关状态管理的功能需求。

## 功能点

### 1. 下拉刷新（Pull-to-Refresh）

- **触发方式**：用户在首个卡片顶部向下滑动手势触发；进入页面时自动触发一次初始刷新。
- **Header**：刷新过程中在顶部展示刷新指示器（Refresh Header）。
- **成功**：全量替换当前所有卡片为新数据，分页页码重置，`hasMore` 按服务端返回更新。
- **失败**：保留现有卡片内容不丢失，弹出刷新失败提示。
- **并发控制**：若当前已有正在进行的刷新或加载更多，再次触发下拉刷新时直接丢弃，等待前一次完成后再由用户手动触发。

### 2. 滑动加载更多（Load More）

- **触发方式**：用户滑动到剩余 **3 个**卡片时，静默自动触发加载更多。
- **Footer**：加载过程中在底部展示加载指示器（Loading Footer）；无更多内容时展示"没有更多内容" Footer。
- **成功**：在现有卡片列表末尾追加新数据，分页页码 +1，`hasMore` 按服务端返回更新。
- **失败（网络/服务器异常）**：保留现有卡片内容，弹出加载失败提示。
- **无更多内容**：当服务端返回空列表时，`hasMore = false`，底部 Footer 变为"没有更多内容"，不再触发加载。
- **并发控制**：若当前已有正在进行的刷新或加载更多，再次触发加载更多时直接丢弃，等待前一次完成。

### 3. 通用并发规则

- 任何时刻，刷新与加载更多 **互斥且不能并发**。
- 若正在刷新或加载中，新的刷新/加载更多请求均直接丢弃，不进入队列。
- 前一次请求完成后，用户可再次手动触发（下拉或滑动）。

### 4. 错误提示区分

| 场景           | 提示方式                          |
|--------------|-------------------------------|
| 刷新失败         | Toast：刷新失败，请重试                |
| 加载更多失败（网络异常） | Toast：加载失败，请重试                |
| 无更多内容        | 不弹 Toast，底部 Footer 展示"没有更多内容" |

### 5. 状态字段

| 字段     | 含义                        |
|--------|---------------------------|
| 卡片列表   | 当前展示的 Feed 卡片列表           |
| 是否刷新中  | 是否正在下拉刷新                  |
| 是否加载中  | 是否正在加载更多                  |
| 是否还有更多 | 是否还有更多内容可加载               |
| 当前页码   | 当前分页页码（刷新重置为 0，加载更多成功 +1） |
| 错误信息   | 当前错误信息（可选，用于调试或 UI 展示）    |

## 测试覆盖需求

1. **初始状态**
    - 卡片列表为空，不处于加载或刷新状态，无错误信息，默认可加载更多。

2. **刷新事件触发刷新状态**
    - 触发刷新后进入刷新中状态，完成后退出刷新状态并填充数据。

3. **刷新成功全量替换数据**
    - 刷新成功后，卡片列表被新数据全量替换，页码重置，是否还有更多按返回更新。

4. **刷新失败保留现有内容**
    - 已有数据时刷新失败，卡片列表保持不变，刷新状态结束，弹出刷新失败提示。

5. **加载更多追加数据**
    - 加载更多成功后，新数据追加到卡片列表末尾，页码递增。

6. **滑动到剩余 3 个卡片时静默触发加载更多**
    - 当滑动到卡片列表末尾还剩 3 个时自动触发加载更多；剩余超过 3 个时不触发。

7. **加载更多到最后一页**
    - 持续加载更多直到服务端返回空列表，此时"是否还有更多"为 false，不再触发加载。

8. **并发触发时丢弃重复请求**
    - 刷新进行中时再次触发刷新或加载更多，新请求被丢弃。
    - 加载更多进行中时再次触发加载更多或刷新，新请求被丢弃。

9. **加载更多失败保留现有内容**
    - 加载更多失败时，卡片列表保持不变，加载状态结束，弹出加载失败提示。
